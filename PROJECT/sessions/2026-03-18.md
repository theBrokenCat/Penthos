**Rutas implementadas:**
- `GET/POST /setup` — primer admin (sin auth)
- `GET /login` — autenticación Auth.js
- `GET /` — dashboard principal
- `GET /audits` — lista de auditorías con filtros
- `GET /audits/[id]` — detalle con tabs (Resumen, Findings, Endpoints, HITL, Config, Informes)
- `GET /agents` — monitor en tiempo real (SSE)
- `GET /settings` — configuración usuario + sección admin

## Fixes aplicados

1. **Directorio `\(dashboard\)` con backslashes literales** — Artefacto de un shell escaping incorrecto durante la generación. Eliminado con `rm -rf`.

2. **Conflicto de rutas Next.js 15 Turbopack** — `/api/hitl/[auditId]` y `/api/hitl/[id]/approve` usaban nombres distintos al mismo nivel (`auditId` vs `id`). Solución: mover a `/api/hitl/audit/[auditId]`.

3. **Servidor arrancado en modo dev** — `npx next dev -p 4000` desde `webapp/src/`.

## Cómo arrancar el portal

```bash
cd openclaw-pentester/webapp/src
npx next dev -p 4000
# → http://localhost:4000
```
