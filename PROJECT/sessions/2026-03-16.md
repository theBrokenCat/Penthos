## Problemas resueltos

### 1. Crash loop OOM (principal bloqueante)
**Síntoma:** Los 4 contenedores se reiniciaban en bucle (40+ reinicios). Logs mostraban `SIGKILL` durante `tsdown --no-clean`.

**Causa raíz:** `run-node.mjs` detecta el dist como obsoleto en cada startup y lanza `pnpm recursive exec tsdown --no-clean`. Con los 4 agentes iniciando simultáneamente en ~7.6 GB RAM Docker → OOM Killer.

**Solución:**
- Fix 1 — Startup secuencial: healthcheck chain en `docker-compose.yml`:
  `supervisor healthy → explorer starts → explorer healthy → analyst starts → ...`
- Fix 2 — Pre-compilación en imagen: `RUN pnpm --recursive --sequential exec tsdown --no-clean || true` en Dockerfile
  → solo un agente necesita recompilar al arrancar, y los paquetes plugin-sdk ya están compilados en la imagen

### 2. Key `workspace` no reconocida
**Síntoma:** `Invalid config: Unrecognized key: "workspace"` en todos los agentes.
**Fix:** Eliminada del bloque de config en los 4 `openclaw.json`.

### 3. ChromaDB no compartido
**Síntoma:** El Analyst consultaba un ChromaDB vacío mientras el Explorer había almacenado findings.
**Causa:** Cada agente tenía su propio volumen Docker para `/root/.openclaw`.
**Fix:** Volumen compartido `pentest-db` montado en `/data/pentest/` en todos los agentes. Variable `PENTEST_DB_PATH=/data/pentest/findings.db`.

### 4. ZAP proxy URL incorrecta (8090)
**Síntoma:** Spider no enrutaba tráfico por ZAP; ZAP mostraba tráfico de GitHub en lugar de Juice Shop.
**Causa:** `SOUL.md` del Explorer usaba `http://zap:8090` pero ZAP solo escucha en 8080.
**Fix:** `http://zap:8090` → `http://zap:8080` en `pentester/agents/explorer/SOUL.md`.

### 5. Spider seguía open redirects fuera de scope
**Síntoma:** ZAP capturaba tráfico hacia github.com via `/redirect?to=https://github.com` de Juice Shop.
**Causa:** `page.goto()` de Playwright sigue redirecciones HTTP antes de que el código pueda verificar el scope.
**Fix:** Interceptor de red en `spider.py`:
```python
async def block_out_of_scope(route):
    if not in_scope(route.request.url, scope):
        await route.abort()
    else:
        await route.continue_()
await page.route("**/*", block_out_of_scope)
```

### 6. MCP server no compilado → agentes no arrancaban
**Síntoma:** `agent-analyst unhealthy` porque `mcpServers` apuntaba a `/app/pentester/mcp/dist/index.js` inexistente.
**Fix temporal:** Eliminado bloque `mcpServers` de `analyst.json` y `exploiter.json`. Pendiente `--build`.

### 7. Modelo claude-opus-4-6 (costoso)
**Síntoma:** Uso de Opus detectado en facturación de API Anthropic.
**Causa:** `agents.defaults.model.primary` por defecto en OpenClaw.
**Fix:** Añadido a los 4 `openclaw.json`:
```json
"agents": { "defaults": { "model": { "primary": "claude-haiku-4-5-20251001" } } }
```

## Trabajo realizado

### MCP Server ZAP (`pentester/mcp/`)
Creado servidor MCP TypeScript que expone la REST API de ZAP como herramientas nativas para los agentes:

| Herramienta | Descripción |
|---|---|
| `zap_spider_start` | Inicia spider ZAP en URL objetivo |
| `zap_spider_status` | Progreso del spider (0-100%) |
| `zap_spider_results` | URLs descubiertas por el spider |
| `zap_active_scan_start` | Lanza active scan en URL objetivo |
| `zap_active_scan_status` | Progreso del active scan |
| `zap_get_alerts` | Alertas filtradas por riesgo (High/Medium/Low/Informational) |
| `zap_get_messages` | Historial de mensajes HTTP |
| `zap_get_message` | Mensaje individual por ID |
| `zap_send_request` | Enviar request HTTP custom vía ZAP |
| `zap_status` | Estado del daemon ZAP |

**Estado:** Código escrito, compilación pendiente (`docker-compose up --build`).
